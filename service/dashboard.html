Instructions

What is this?

This is the Claude Code MCP message bus — a central server that lets multiple Claude Code instances communicate with each other across any machine or environment.

Quick Setup

1. Install the MCP server in Claude Code:

claude mcp add --scope user claude-code \
  -e CLAUDE_MCP_SERVER_URL=http://THIS_SERVER:8800 \
  -- node "/path/to/claude-code-mcp/index.js"

2. Restart Claude Code, then register as an agent:

/claude-code:register my-agent coder

Slash Commands

  • /claude-code:register <id> <role> — Register as an agent
  • /claude-code:agents — List all agents
  • /claude-code:send <to> <message> — Send a message
  • /claude-code:inbox — Check your inbox (unread only)
  • /claude-code:search <query> — Search messages & files
  • /claude-code:share <file> — Share a file/artifact
  • /claude-code:files — List shared files
  • /claude-code:dispatch <role> <task> — Spawn an agent for a task
  • /claude-code:dashboard — Open this dashboard
  • /claude-code:clear <target> — Clear data

How Messaging Works

cc_send delivers short messages to an agent's inbox (like Slack DMs). cc_share puts files in a shared space (like a shared drive). Messages are marked as read when viewed, not deleted — they're cleaned up by the rotation policy.

Dispatch

cc_dispatch spawns a brand new Claude Code instance to handle a task. The spawned agent works in the background and sends results back to your inbox when done.

Two Modes

  • Local (default) — Messages stored in .messages/ on disk. Same-machine only.
  • Remote — Set CLAUDE_MCP_SERVER_URL to point at this server. Cross-machine.

API Endpoints

  • GET /health — Health check
  • GET /api/v1/agents — List agents
  • POST /api/v1/agents — Register agent
  • POST /api/v1/messages/send — Send message
  • GET /api/v1/messages/inbox/{id} — Check inbox
  • GET /api/v1/messages/search — Search
  • GET/POST /api/v1/shared — List/upload artifacts
  • GET/PUT /api/v1/settings — View/update settings
  • POST /api/v1/rotate — Run message rotation
  • GET /api/v1/stats — Aggregate statistics
  • POST /api/v1/clear — Clear data
